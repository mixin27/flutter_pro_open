# Contributing

Thanks for considering contributing!

### 1. Filing Issues

- Use the correct template for bugs, features, questions, docs, or tasks.
- Search for existing issues first.

### 2. Working on Code

- Fork and create a descriptive branch (e.g., `fix/42-crash`, `feature/15-dark-mode`).
- Reference the issue in your commits/pr (e.g., `Fixes #42`).
- Ensure code is formatted (`flutter format .`), analyzed (`flutter analyze`), and tested (`flutter test`).

### 3. Submitting Pull Requests

- Explain your changes using the PR template.
- Complete all checklist items.
- Wait for CI to pass and review from maintainers.

### 4. Discussions & Conduct

- Use Discussions for open questions.
- Be positive and respectful (see [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md)).
